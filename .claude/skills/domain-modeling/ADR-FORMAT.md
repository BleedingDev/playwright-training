# ADR format

Create ADRs under `docs/adr/` only when the decision is hard to reverse, surprising without context, and based on a genuine trade-off.

```md
# {Short decision title}

{One to three sentences describing the context, decision, and reason.}
```

Use sequential filenames such as `0001-provider-boundary.md`. Add status, considered options, or consequences only when they add concrete value.
