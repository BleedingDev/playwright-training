# SídloFlow agent rules

- Autoritatívny doménový vstup je `PRODUCT_CONTRACT.md`.
- Test Brief pre hlavný flow je `specs/address-change.md`, keď na branche existuje.
- Spustenie aplikácie: `mise run app:up`.
- Reset dát: `mise run app:reset`.
- Diagnostika prostredia: `mise run doctor`.
- Kanonický test po zmene: `mise run verify`.
- Funkčné E2E: `mise run test:e2e`.
- Docker E2E: `mise run test:e2e:docker`.
- Smoke: `mise run test:smoke`.
- Visual: `mise run test:visual`.
- Postupuj: explore -> clarify -> brief -> test -> prove -> report.
- Najprv pochop chránený používateľský zámer.
- Pred implementáciou objasni nejednoznačnosti.
- Uprednostni role, label a text locators pred CSS selektormi.
- Assertions smeruj na pozorovateľný výsledok pre používateľa.
- Používaj web-first assertions.
- Nepoužívaj pevné sleeps ani `waitForTimeout`.
- Pri zlyhaní najprv diagnostikuj trace, screenshot, log a DOM.
- Aplikáciu nemeň iba preto, aby test zozelenel.
- Mockuj explicitnú boundary, nie vnútornú implementáciu.
- `page.route` mockuje hranicu pre jeden Playwright test.
- MSW mockuje rovnakú React -> Laravel hranicu v lokálnom browseri.
- Laravel -> RÚIAN adapter overuj samostatne.
- Nevyberaj svojvoľne subset funkčných testov podľa diffu.
- `verify` vždy spúšťa pevnú funkčnú sadu.
- Visual suite zostáva oddelená.
- Nikdy automaticky neaktualizuj screenshot baseline.
- Baseline generuj iba v pripnutom Playwright Docker image.
- V reporte uveď spustené príkazy a ich výsledky.
- Uveď cesty k reportu, trace, screenshotom a diffom.
